---

### ✨ Purpose of This Template

This is the **canonical format** for all NorthStar client-facing offer reports. It defines the exact structure, sections, depth of analysis, and formatting standards for any role that has cleared the hard disqualifier gate and is being presented to the client for serious consideration.

Every report produced from this template answers one question with evidence: **"Is this specific offer a genuine, provable match for this specific client — and exactly why?"**

A valid dossier under this template must include:
• The complete job listing text (verbatim, so evidence is auditable)
• A requirement-by-requirement match matrix against the client's confirmed skills
• An honest pros/cons assessment with no invented claims
• A risk audit with severity ratings and specific mitigations
• A gate decision with a clear GO / CONDITIONAL GO / HOLD / REJECT outcome
• A concrete execution support plan if the gate allows application

---

### ⚠️ Hard Disqualifier Pre-Check (Must Pass Before This Template Is Used)

**This template is ONLY used for roles that pass ALL hard blocks.**  
If ANY of the following apply, the role is Tier D and this template is not used — the role goes directly to the rejection log.

| Hard Block | Trigger | Decision |
|:---|:---|:---|
| **Portuguese-mandatory language** | Listing requires fluent/professional Portuguese; no English-only option | Tier D — Auto-reject |
| **Non-English mandatory language** | Requires German / French / Dutch / Italian / Spanish as working language | Tier D — Auto-reject |
| **Seniority mismatch** | Role requires Senior / Lead / Principal / Manager / Head Of | Tier D — Auto-reject |
| **Years required ≥ 4** | Listing states 4+ years minimum required experience | Tier D — Auto-reject |
| **Role family off-target** | DevOps / Java Developer / Sales / Pure Call Center / CCTV / Telecom / Medical | Tier D — Auto-reject |

💡 **Exception rule — language:** If the mandatory language phrase is qualified by "nice to have", "is a plus", "preferred", or "valorizado" — role proceeds with a -10 soft penalty only. This exception must be documented in Section 3.

## Standards & Protocol Notes

---

### 🔍 What Makes This Template Industry-Standard

This structure is directly modelled on best practices from the top talent placement and ATS platforms catalogued in `research/agency_benchmarks/agency_benchmark_db.json`:

**Greenhouse Scorecard Architecture** — The requirement-to-client match matrix (Section 4) mirrors how Greenhouse structures interview scorecards: each criterion is listed explicitly, evidence is evaluated independently, and a confidence rating is applied. This prevents the "vibes-based" assessment that leads to false positives.

**Workable Structured Interview Kits** — The gate system (Section 6 / Application Readiness Gate) mirrors Workable's structured hiring gates: no candidate moves to the next phase without explicitly passing each gate. "CONDITIONAL GO" corresponds to Workable's "proceed with conditions" gate state.

**Randstad RiseSmart Outplacement Framework** — The Execution Support section (Section 7) mirrors the lifecycle support model: the agency doesn't just identify the opportunity, it prepares the client for every step from application to follow-up. This is the difference between sourcing and placement.

**Robert Half IT Resume Benchmarks** — Requirement language in Section 4 must use the same vocabulary as IT job postings. Evidence descriptions should be phrased in recruiter-readable language, not developer shorthand.

---

### ⚠️ Mandatory Rules for Every Report

**Rule 1 — Verbatim listing text is mandatory.** Section 2 must contain the exact employer language. Reports without source text are unauditable and cannot be defended to the client.

**Rule 2 — Evidence must be sourced.** Every entry in the match matrix must cite where the evidence comes from — skills ledger, CV, assessment, or client-stated. "Probably" is not a source. Label it ⚠️ Probable and mark the action.

**Rule 3 — No invented qualifications.** If a skill is not in the ledger and has not been tested, it is ❓ Unknown. Do not promote it to ⚠️ Probable without a specific logical basis from the career timeline.

**Rule 4 — Hard blocks override all scoring.** If any hard disqualifier triggers after the report is started, the report is abandoned, the role is reclassified Tier D, and this is logged in the rejection record.

**Rule 5 — Gate decision must be explicit.** Every report ends with one of four decisions: `GO`, `CONDITIONAL GO`, `HOLD`, or `REJECT`. No report is valid without a gate decision.

**Rule 6 — Language gate is always active.** Even for English-heavy listings, the working language must be confirmed. A listing in English that mentions "Portuguese office environment" still triggers a language flag.
